
## 2026-02-27 — 014-m1-core-integration-map-into-read-tool-p

- **Batch issues work well for tightly-coupled features**: Grouping #004 (integration), #005 (prompt), and #006 (tests) into a single batch (#014) avoided context-switching overhead — the integration code, prompt docs, and tests all informed each other during development.
- **Map cache is dead simple and that's fine**: A plain `Map<string, { mtimeMs, map }>` with no LRU or size limits is totally adequate for a single-session extension. The YAGNI call on cache eviction was correct — no issues observed even with 10K+ line fixture files.
- **Silent error swallowing is the right pattern for "enhancement" features**: The map is optional enrichment on top of hashlines. Making every map-related failure a silent catch (returning null or unchanged output) means the read tool's reliability is decoupled from the readmap library's reliability. This was validated when tests confirmed zero error propagation paths.
- **Fixture generation for large files needs care**: The 10,680-line `large.ts` fixture has realistic-looking code (EventEmitter, DatabaseConnection, TaskRunner, DataProcessor classes) which makes integration tests meaningful — they check for real symbol names, not just line counts. Padding with comments would have been faster but less useful.
- **The "truncated + no offset + no limit" condition is the key design insight**: Only appending maps when the user hasn't already specified a targeted read prevents redundant maps when someone is already doing focused reads. This came from the spec phase and proved to be the right abstraction boundary.
- **Test structure mirrors acceptance criteria 1:1**: Having test files map directly to AC groups (map-cache.test.ts → AC-6, read-integration.test.ts → AC-7/8) made verification straightforward — each test file is independently verifiable against its criteria.
- **`formatFileMapWithBudget()` budget enforcement is inherited for free**: The readmap formatter already handles tiered detail reduction (Full → Compact → Minimal → Outline → Truncated), so the read tool integration didn't need any budget logic — just call the function and trust the output fits.
